# test-driven-development

## Overview
TDD: כותבים את הבדיקה לפני הקוד. צופים בה נכשלת. כותבים את הקוד המינימלי כדי שתעבור. **אם לא ראית את הבדיקה נכשלת — אתה לא יודע אם היא בודקת את הדבר הנכון.**

## Source path

## When to invoke
תמיד: פיצ'רים חדשים, תיקוני באגים, refactoring, שינויי התנהגות. חריגים (צריכים לוודא עם המשתמש): prototypes זריקים, קוד generated, קבצי config.
